# API Reference

The exit1.dev API provides programmatic access to your monitoring checks, history, and statistics.

> **Note:** API access is available on all plans.

## Base URL

All API requests are made to:

```
https://us-central1-exit1-dev.cloudfunctions.net/publicApi
```

## Conventions

- **REST architecture** - Resources are accessed via standard HTTP methods
- **JSON responses** - All responses are returned in JSON format
- **API versioning** - All endpoints are prefixed with `/v1/`
- **Scoped access** - Write and delete operations require [API key scopes](/api-reference/scopes)

## Available Endpoints

### Read Endpoints

| Endpoint | Description |
|---|---|
| `GET /v1/public/checks` | [List all checks](/api-reference/list-checks) |
| `GET /v1/public/checks/{id}` | [Get a single check](/api-reference/get-check) |
| `GET /v1/public/checks/{id}/history` | [Get check history](/api-reference/check-history) |
| `GET /v1/public/checks/{id}/stats` | [Get uptime statistics](/api-reference/check-stats) |

### Write Endpoints

| Endpoint | Scope Required | Description |
|---|---|---|
| `POST /v1/public/checks` | `checks:write` | [Create a new check](/api-reference/create-check) |
| `PATCH /v1/public/checks/{id}` | `checks:write` | [Update an existing check](/api-reference/update-check) |
| `DELETE /v1/public/checks/{id}` | `checks:delete` | [Delete a check](/api-reference/delete-check) |
| `POST /v1/public/checks/{id}/toggle` | `checks:write` | [Enable or disable a check](/api-reference/toggle-check) |

## Quick Start

1. [Create an API key](/api-reference/authentication) with the appropriate [scopes](/api-reference/scopes)
2. Include the key in the `X-Api-Key` header
3. Make your first request:

```bash
curl -H "X-Api-Key: ek_live_your_api_key_here" \
  https://us-central1-exit1-dev.cloudfunctions.net/publicApi/v1/public/checks
```
